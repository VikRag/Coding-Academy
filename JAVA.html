Java Programming Language

Basics

Core concepts of Java programming including variables, operators, control flow, loops, arrays, and basic syntax. These form the foundation for writing any Java program.

Introduction & Syntax

Java is a platform-independent language developed by Sun Microsystems (now Oracle) in 1995. It follows the "Write Once, Run Anywhere" principle thanks to the JVM, which compiles Java code into bytecode that can run on any device with a JVM.

Basic syntax requires every program to be in a class, and execution begins in the main method. Java is case-sensitive and uses semicolons to end statements.

  • Introduction: Java is platform-independent and follows “Write Once, Run Anywhere.” It's widely used for web, mobile, and enterprise applications.
  • Syntax: Programs are inside classes; execution starts at the main method. Comments can be single-line (//) or multi-line (/* */).
  • Output: Use System.out.println() to print to the console. For formatted output, use System.out.printf().
public class HelloWorld {
    // This is the main method where execution starts
    public static void main(String[] args) {
        // Print a message to the console
        System.out.println("Hello, World!");  // Outputs: Hello, World!
        
        // Formatted output example
        System.out.printf("Java version: %s\n", "21");  // Outputs: Java version: 21
    }
}

In the code above, the main method is static so it can be called without creating an object. The args parameter allows command-line arguments.

Variables & Operators

Variables store data and must be declared with a type. Java has primitive types like int, double, char, boolean, and reference types like String.

Operators include arithmetic (+, -, *, /, %), comparison (==, !=, >, <), logical (&&, ||, !), and assignment (=, +=, etc.).

  • Variables: Store numbers, text, and boolean values. Must be initialized before use in some contexts.
  • Operators: Perform arithmetic, comparison, and logical operations. Operator precedence follows PEMDAS.
// Declare and initialize variables
int x = 5;  // Integer variable
double y = 10.5;  // Floating-point variable
boolean isTrue = true;  // Boolean variable
String text = "Java";  // String variable

// Arithmetic operators
int sum = x + (int)y;  // Cast y to int, sum = 15
double product = x * y;  // product = 52.5

// Comparison and logical operators
if (x > 0 && isTrue) {  // Checks if x is positive AND isTrue is true
    System.out.println("Positive and true");  // Outputs if condition met
}

// Assignment operators
x += 3;  // x now equals 8

Note: Java is strongly typed, so implicit type conversions are limited. Use casting for explicit conversions.

Path to Expertise: Practice declaring variables of all primitive types and using operators. Experiment with type casting. W3Schools covers this in Variables, Data Types, Type Casting, Operators, Strings, Math, and Booleans. Do the exercises and use the online compiler to test variations.

Control Flow & Loops

Control flow allows decision-making and repetition. If-else for conditions, switch for multiple cases, loops for iteration.

  • If / Else: Conditional statements for decision-making. Can be nested or use else-if chains.
  • Loops: Repeat code using for (counted), while (condition-based), or do-while (post-check).
  • Switch: Handles multiple conditions based on a value.
// If-else example
int num = 10;
if (num > 0) {
    System.out.println("Positive");  // Executes if num > 0
} else if (num < 0) {
    System.out.println("Negative");
} else {
    System.out.println("Zero");
}

// For loop: iterates a fixed number of times
for (int i = 0; i < 5; i++) {  // i from 0 to 4
    System.out.println(i);  // Outputs 0 to 4
}

// While loop: checks condition before execution
int count = 0;
while (count < 3) {
    System.out.println("Count: " + count);
    count++;  // Increment to avoid infinite loop
}

// Do-while: executes at least once
do {
    System.out.println("Executes once");
} while (false);  // Condition false, but runs once

// Switch statement
switch (num) {
    case 10:
        System.out.println("Ten");
        break;  // Exit switch
    default:
        System.out.println("Other");
}

Always include break in switch cases to prevent fall-through. Loops can use break to exit early or continue to skip iterations.

Path to Expertise: Build simple programs like calculators or number guessers using if-else and loops. W3Schools sections: If...Else, Switch, While Loop, For Loop, Break/Continue. Practice exercises and "How To" examples like Even/Odd or Positive/Negative checks.

Strings & Arrays

Strings are immutable objects with methods like length(), substring(), concat(). Arrays are fixed-size collections of the same type.

  • Strings: Represent text and have useful built-in methods like toUpperCase(), equals().
  • Arrays: Store multiple values of the same type. Can be multi-dimensional.
// String operations
String greeting = "Hello";  // Create string
String full = greeting.concat(" World");  // "Hello World"
int len = full.length();  // 11
String upper = full.toUpperCase();  // "HELLO WORLD"
boolean equals = greeting.equals("hello");  // false (case-sensitive)

// Array declaration and initialization
int[] numbers = {1, 2, 3, 4, 5};  // Fixed-size array
String[] fruits = new String[3];  // Array of size 3
fruits[0] = "Apple";  // Assign value

// Accessing and looping through array
for (int num : numbers) {  // Enhanced for loop
    System.out.println(num);  // Outputs 1 to 5
}

// Multi-dimensional array
int[][] matrix = {{1, 2}, {3, 4}};  // 2x2 matrix
System.out.println(matrix[1][0]);  // 3

Strings are compared with equals() not ==, as == checks reference. Arrays can throw ArrayIndexOutOfBoundsException if accessed out of bounds.

Path to Expertise: Manipulate strings (e.g., reverse, count vowels) and arrays (sort, find max/min). W3Schools: Strings, Arrays. Use "How To" for tasks like Reverse String, Palindrome Check, Sort Array.

Methods

Methods are reusable blocks of code that can take parameters and return values. They promote code modularity and reusability.

Methods & Parameters

Methods are defined with access modifier, return type, name, and parameters. They can be static (class-level) or instance (object-level).

  • Methods: Perform specific tasks and improve code organization. Can return values or be void.
  • Parameters: Pass data into methods. Can have default values in some contexts, but not in standard Java methods.
  • Varargs: Allow variable number of arguments using ...
// Simple method with parameters
public static void greet(String name) {  // void return type
    System.out.println("Hello, " + name);  // Use parameter
}

// Method with return value
public static int add(int a, int b) {
    return a + b;  // Return sum
}

// Varargs example
public static void printNumbers(int... nums) {  // Variable arguments
    for (int num : nums) {
        System.out.println(num);
    }
}

// Calling methods
public static void main(String[] args) {
    greet("Alice");  // Outputs: Hello, Alice
    int sum = add(5, 3);  // sum = 8
    printNumbers(1, 2, 3);  // Outputs 1,2,3
}

Parameters are passed by value in Java. For objects, this means the reference is copied.

Path to Expertise: Create utility methods for common tasks. W3Schools: Methods, Method Parameters, Scope.

Overloading & Recursion

Method overloading allows multiple methods with the same name but different signatures. Recursion is when a method calls itself.

  • Overloading: Same method name with different parameters. Resolved at compile time.
  • Recursion: Method calling itself with a base case to prevent stack overflow.
// Overloading examples
public static int add(int a, int b) {  // Two ints
    return a + b;
}
public static double add(double a, double b) {  // Two doubles
    return a + b;
}
public static int add(int a, int b, int c) {  // Three ints
    return a + b + c;
}

// Recursion example: Factorial
public static int factorial(int n) {
    if (n <= 1) {  // Base case
        return 1;
    }
    return n * factorial(n - 1);  // Recursive call
}

// Usage
public static void main(String[] args) {
    System.out.println(add(2, 3));  // 5 (int version)
    System.out.println(add(2.5, 3.5));  // 6.0 (double version)
    System.out.println(factorial(5));  // 120
}

Overloading improves readability. Recursion is useful for problems like tree traversal but can be inefficient for large depths.

Path to Expertise: Implement recursive solutions (factorial, Fibonacci) and overloaded methods. W3Schools: Method Overloading, Recursion. Practice with "How To" like Factorial or Fibonacci.

Object-Oriented Programming

Java models real-world objects using classes and objects. Key OOP principles include encapsulation, inheritance, polymorphism, and abstraction.

Classes & Constructors

Classes are blueprints defining fields and methods. Constructors initialize objects and can be overloaded.

  • Classes: Blueprints for creating objects. Can have fields, methods, constructors.
  • Constructors: Initialize new objects. Default constructor if none defined.
  • Encapsulation: Use private fields with public getters/setters.
public class Car {
    // Private fields for encapsulation
    private String model;
    private int year;
    
    // Default constructor
    public Car() {
        this.model = "Unknown";
        this.year = 0;
    }
    
    // Parameterized constructor
    public Car(String model, int year) {
        this.model = model;  // 'this' refers to instance field
        this.year = year;
    }
    
    // Getter method
    public String getModel() {
        return model;
    }
    
    // Setter method
    public void setYear(int year) {
        this.year = year;
    }
}

// Usage
public static void main(String[] args) {
    Car myCar = new Car("Tesla", 2023);  // Create object
    System.out.println(myCar.getModel());  // Tesla
}

'this' keyword distinguishes instance variables from parameters. Constructors can call each other using this().

Path to Expertise: Model real-world entities (e.g., BankAccount, Person). W3Schools: Classes/Objects, Constructors, Modifiers, Encapsulation.

Inheritance & Polymorphism

Inheritance allows subclasses to inherit from superclasses. Polymorphism enables objects to be treated as instances of their parent class.

  • Inheritance: Reuse code from other classes using extends. Supports single inheritance.
  • Polymorphism: Methods behave differently based on object type. Achieved via overriding.
  • Super: Call superclass methods or constructors.
// Superclass
public class Vehicle {
    public void drive() {
        System.out.println("Driving vehicle");
    }
}

// Subclass with inheritance
public class ElectricCar extends Vehicle {  // Inherits from Vehicle
    @Override  // Annotation for overriding
    public void drive() {  // Polymorphic method
        System.out.println("Driving electrically");
    }
    
    public void charge() {  // New method
        System.out.println("Charging");
    }
}

// Usage demonstrating polymorphism
public static void main(String[] args) {
    Vehicle car = new ElectricCar();  // Upcasting
    car.drive();  // Outputs: Driving electrically (runtime polymorphism)
    // car.charge(); // Not accessible via Vehicle reference
}

@Override ensures correct overriding. Final methods/classes prevent inheritance/overriding.

Path to Expertise: Build class hierarchies (e.g., Animal > Dog/Cat). W3Schools: Inheritance, Polymorphism, Inner Classes.

Abstraction & Interfaces

Abstraction hides details using abstract classes or interfaces. Interfaces define contracts with default methods since Java 8.

  • Abstraction: Hide complex implementation details using abstract classes (can have concrete methods).
  • Interfaces: Define methods classes must implement. Supports multiple inheritance.
  • Default methods: Provide implementation in interfaces.
// Abstract class
abstract class Shape {
    abstract double area();  // Abstract method
    public void display() {  // Concrete method
        System.out.println("Displaying shape");
    }
}

// Interface
interface Drivable {
    void drive();  // Abstract method
    default void stop() {  // Default method
        System.out.println("Stopping");
    }
}

// Implementing class
public class Circle extends Shape implements Drivable {
    double radius;
    public Circle(double radius) {
        this.radius = radius;
    }
    
    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public void drive() {
        System.out.println("Driving circle?");  // Hypothetical
    }
}

// Usage
public static void main(String[] args) {
    Circle c = new Circle(5);
    System.out.println(c.area());  // ~78.54
    c.drive();  // Driving circle?
    c.stop();  // Stopping
}

Abstract classes can have constructors; interfaces cannot (pre-Java 8). Use interfaces for multiple behaviors.

Path to Expertise: Design systems with abstract classes and interfaces (e.g., shapes, vehicles). W3Schools: Abstraction, Interface, Enum.

Advanced Topics

Advanced features of Java for robust and interactive applications, including input handling, collections, error management, concurrency, and I/O.

User Input & Collections

User input uses Scanner for console reading. Collections framework provides dynamic data structures like List, Set, Map.

  • User Input: Use Scanner class to read input from users. Supports nextInt(), nextLine(), etc.
  • Collections: Store groups of objects using ArrayList (resizable array), HashSet (unique elements), HashMap (key-value pairs).
  • Generics: Type-safe collections, e.g., ArrayList<String>.
import java.util.Scanner;  // Import for input
import java.util.ArrayList;  // Import for List
import java.util.HashMap;  // Import for Map

// User input
Scanner scanner = new Scanner(System.in);  // Create scanner
System.out.print("Enter name: ");
String name = scanner.nextLine();  // Read string
System.out.print("Enter age: ");
int age = scanner.nextInt();  // Read int

// Collections: ArrayList
ArrayList fruits = new ArrayList<>();  // Generic list
fruits.add("Apple");  // Add element
fruits.add("Banana");
fruits.remove(0);  // Remove by index
for (String fruit : fruits) {  // Iterate
    System.out.println(fruit);  // Outputs Banana
}

// HashMap
HashMap scores = new HashMap<>();
scores.put("Alice", 90);  // Add key-value
int aliceScore = scores.get("Alice");  // 90

Close scanner with scanner.close() to prevent resource leaks. Collections are in java.util package.

Path to Expertise: Build programs with user interaction and data storage. W3Schools: User Input, ArrayList, LinkedList, HashMap, Iterator.

Exceptions & Threads

Exceptions handle runtime errors. Threads enable concurrency for multi-tasking.

  • Exceptions: Handle errors gracefully using try-catch. Can throw custom exceptions.
  • Threads: Execute multiple tasks concurrently using Thread class or Runnable interface.
  • Synchronization: Use synchronized to prevent race conditions.
// Exception handling
try {
    int divide = 10 / 0;  // ArithmeticException
} catch (ArithmeticException e) {  // Catch specific exception
    System.out.println("Error: " + e.getMessage());  // Division by zero
} finally {  // Always executes
    System.out.println("Cleanup");
}

// Custom exception
class MyException extends Exception {
    public MyException(String msg) {
        super(msg);
    }
}

// Throwing exception
public static void checkAge(int age) throws MyException {
    if (age < 18) {
        throw new MyException("Too young");
    }
}

// Thread example
class MyThread extends Thread {
    public void run() {  // Thread execution
        System.out.println("Thread running");
    }
}

// Usage
public static void main(String[] args) {
    MyThread t = new MyThread();
    t.start();  // Start thread
    // Runnable alternative: new Thread(() -> { /* code */ }).start();
}

Checked exceptions must be handled or declared. Threads can use join() to wait for completion.

Path to Expertise: Create multi-threaded apps and robust error handling. W3Schools: Exceptions, Threads, Lambda.

File Handling

File handling for reading/writing data. Use try-with-resources for auto-closing.

  • File Handling: Read and write files for permanent storage using File, FileReader, FileWriter, etc.
  • Buffered I/O: For efficiency, use BufferedReader/Writer.
  • Paths: Java NIO for modern file operations.
import java.io.*;  // Import I/O classes
import java.nio.file.*;  // For NIO

// Writing to file
try (FileWriter writer = new FileWriter("example.txt")) {  // Try-with-resources
    writer.write("Hello, File!");  // Write string
} catch (IOException e) {
    System.out.println("Write error: " + e);
}

// Reading from file
try (BufferedReader reader = new BufferedReader(new FileReader("example.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {  // Read line by line
        System.out.println(line);  // Outputs: Hello, File!
    }
} catch (IOException e) {
    System.out.println("Read error: " + e);
}

// NIO example: Delete file
Path path = Paths.get("example.txt");
Files.deleteIfExists(path);  // Delete if exists

Handle IOExceptions. NIO provides non-blocking I/O and better performance for large files.

Path to Expertise: Build file-based apps (e.g., todo list saver). W3Schools: File Handling, Read/Write/Create/Delete Files.
